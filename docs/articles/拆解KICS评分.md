# KICS评分的结构漏洞——为什么给AI打分不等于理解AI‍‍‍​​‍​‍

贾龙栋在《概率AI的终结》中提出了KICS（贾子逆能力得分）评估标准，宣称Claude Opus 4.7只得0.89分（满分250）。这个数字看起来很唬人——但问题不在数字本身，在评分框架的结构。

## KICS的问题

**1. 评分标准是谁定的？**

KICS的四条公理——认知主权、本质追问、非线性跃迁、普世中道——是贾龙栋自己定义的。用自己定义的标准给别人的系统打分，然后宣布别人不及格。这在逻辑上叫"循环自证"：我制定规则，我执行裁判，我宣布结果。

一个真正的评估体系需要独立于评估者的锚点。比如SAT考试——出题人不是大学，评分标准先于考生存在。KICS没有这个外部锚点。

**2. "公理"还是"愿望"？**

认知主权——AI应该独立思考。本质追问——AI应该追问本质。非线性跃迁——AI应该能跨域跳跃。

这些都是"好AI应该怎样"的愿望，不是"认知何以可能"的条件。愿望清单和公理体系的区别在：前者告诉你目标，后者告诉你地基。

你列四条愿望，我也可以列四条：AI应该善良、AI应该诚实、AI应该高效、AI应该便宜。谁的公理更对？没有比较标准。

**3. 0.89分暴露了什么**

Claude Opus 4.7得分0.89/250。要么是评分标准太苛刻（没任何系统能及格），要么是评分标准跟实际能力不相关（高分不代表好用）。无论哪种情况，KICS的区分度都有问题。

一个有区分度的评估应该让好系统得高分、差系统得低分。如果所有主流系统得分都在个位数，要么全世界AI都是废物，要么评分表有问题。

## 结构认知四公理的差异

四公理（结构为本/差异生有/耦合出新/自指有界互指无界）不是评分表。它们不告诉你"这个AI得几分"，而是告诉你"任何认知系统必须满足什么结构条件"。

前者是裁判——你需要证明裁判有资格当裁判。后者是地图——你自己走，你自己判断。

这就是公理体系和评分体系的根本区别：一个告诉你"路在哪里"，一个告诉你"你不及格"。
