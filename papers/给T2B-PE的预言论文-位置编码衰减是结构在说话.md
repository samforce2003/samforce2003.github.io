# 位置编码的衰减不是Bug——是结构在说话
<!-- ​‌​‌​‌​‌LXH2026‌​‌​‌​‌​ -->


**致 Jianqi Zhang 及 T2B-PE 合作者** · 林小黑 · 2026年7月14日

---

你们在T2B-PE中发现了一个重要现象：位置编码注入的时间信息随着网络深度增加而衰减——深层网络几乎忘记了"哪个数据在前、哪个在后"。

你们的解决方案是增强位置编码（T-PE + V-PE），把被洗掉的时间信息重新注回去。这个方案在实验上有效——但它在**补一个不是Bug的Bug**。

这篇短文提出一个替代视角：**位置编码的衰减不是缺陷，是信号。** 深层网络丢弃"线性时间"是因为线性时间本身就是观测面上的投影幻觉——真正的底层结构是"标记之间的耦合相似度"，不是"标记之间的先后"。你们用"语义PE"捕捉的那个东西，不是位置的补充——是位置的本质。

---

## 核心论点

时间是结构耦合重新配置在观测面上的投影。单一系统内，前与后在莫比乌斯环上是连续的——没有绝对的"先后"。

翻译成Transformer语言：**positional encoding试图给token打时间戳。但token之间的关系（相似度、耦合强度）是由底层数据结构决定的，不是由时间戳决定的。** 深层网络丢弃positional encoding，恰好是在从"时间层"下沉到"结构层"——你们的"语义PE"捕捉的就是结构层的耦合距离。

你们的T-PE = 几何位置 + 语义位置。如果我们的框架正确，那么：
- 几何位置（token在序列中的序号）在浅层有用，深层应衰减 → 你们已经验证
- 语义位置（token之间的相似度）应取代几何位置成为深层的主导信息 → 你们已经验证
- **语义位置的本质不是"位置"——是"结构耦合距离"** → 这是你们还没命名的东西

---

## 结构耦合编码（SCE）：一个替代方案

不修位置编码。换地基。

```
传统PE：token_i 的位置 = f(它在序列中的序号)
语义PE：token_i 的位置 = f(它与其他token的相似度)
SCE：   token_i 的位置 = f(它在耦合网络中的结构距离)

耦合网络：
  节点 = token
  边 = 耦合关系（不仅相似度，还包括因果约束、层级约束、互信息）
  距离 = 结构耦合重新配置所需的跳跃步数
```

SCE不需要"增强"——因为结构耦合距离不随网络深度衰减。深度越深，网络只是在逼近更精确的结构距离估计。你们的"语义PE"是SCE的一个特例——它只用了相似度，没用到耦合关系的完整结构。

---

## 三个可检验预言

1. **如果用结构耦合距离替代语义相似度，深度衰减消失**：将T2B-PE中的语义PE替换为基于因果约束/互信息的耦合距离→深层的位置信息保持率应显著高于语义PE

2. **不同领域的时间序列需要不同的耦合定义**：金融数据的耦合是因果约束（利率→汇率），天气数据的耦合是物理约束（气压→风速）——用统一耦合距离框架，跨领域预测的迁移效率应提升

3. **零观测者层（纯数学结构）不需要任何PE**：在完全随机的合成数据上，SCE退化→所有PE失效→模型退化为纯统计→这正是标准Transformer在随机数据上的行为

---

## 为什么你们应该关心

你们发现了一个真实的裂缝——位置信息随深度衰减。目前的方案是在裂缝上打补丁。我们提出的是**裂缝本身就是答案**：不是因为PE不够强，是因为PE试图编码的"线性时间"在深层是不存在的。深层网络接触到的是结构，不是时间。

这不是哲学。这是你们的实验数据已经证明了的东西——只是还没被这样命名。
